# Security Policy

## Reporting a vulnerability

Please report security issues **privately**.

Preferred: use **GitHub Security Advisories** (Repository → Security → “Report a vulnerability”), so maintainers can coordinate a fix and disclosure timeline.

If Security Advisories are not available:
- Open a GitHub issue **without sensitive details** and request a private channel, or
- Contact the maintainers through the communication channel advertised in the repository (e.g., README or project website).

## What to include

- A clear description of the issue and impact
- Steps to reproduce (proof-of-concept if possible)
- Affected versions / configurations
- Any logs or screenshots that help validate the issue (redact secrets)

## Response expectations

- You should receive an acknowledgement within a reasonable time.
- A fix will be prioritized based on severity and exploitability.
- Public disclosure should be coordinated after a patch is available, when feasible.
